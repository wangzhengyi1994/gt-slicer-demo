# GT Slicer Demo - 菜单交互功能实现

## 任务
为顶部菜单栏的所有菜单项添加点击交互功能。当前状态: 菜单可以展开,但点击菜单项没有任何反应。

## 需要实现的交互

### 通用规则
- 点击菜单项后,菜单自动关闭
- 有实际功能的操作执行功能,没有实际功能的显示 toast 提示(底部居中弹出,2秒消失)
- Toast 样式: 暗色半透明背景,白色文字,圆角,出现/消失有动画

### 文件菜单 (F)
- **新建项目** (Ctrl+N): 弹出确认对话框 "是否新建项目? 当前未保存的更改将丢失"
- **打开文件** (Ctrl+O): 触发一个 file input 让用户选择文件(支持 .stl, .obj, .3mf),选择后 toast "已加载: 文件名"
- **最近打开**: 展开子菜单显示 3 个假的最近文件(模拟数据)
- **保存项目** (Ctrl+S): toast "项目已保存"
- **另存为** (Ctrl+Shift+S): toast "项目已另存为"
- **导入** (Ctrl+I): 同打开文件
- **导出** (Ctrl+E): 如果已有 exportModal 弹窗就打开它,否则 toast "导出完成"
- **退出** (Ctrl+Q): 弹出确认 "确定要退出吗?"

### 编辑菜单 (E)
- **撤销/重做**: toast "撤销/重做" (demo 无实际数据)
- **全选**: toast "已全选所有模型"
- **自动排列**: toast "已自动排列模型"
- **复制所选**: toast "已复制所选模型"
- **删除所选**: toast "已删除所选模型"
- **清空打印平台**: 已有 data-confirm,确保弹窗能正常弹出
- **重置所有模型位置**: toast "已重置所有模型位置"
- **重置所有模型旋转**: toast "已重置所有模型旋转"
- **编组/合并/取消编组**: toast 对应操作名

### 视图菜单 (V)
- **视角**: 子菜单展开(保持现有)
- **相机视图**: 子菜单展开(保持现有)
- **显示网格**: toggle 行为,菜单项前加 ✓ 标记,toast "网格已显示/隐藏"
- **显示坐标轴**: toggle 行为,同上
- **全屏** (F11): 调用 document.documentElement.requestFullscreen()

### 设置菜单 (S)
- 挤出机子菜单已有结构,点击 **喷嘴尺寸** 时切换 ✓ 标记并 toast "已切换喷嘴: 0.4mm"
- 点击 **耗材品牌** 时切换 ✓ 标记并 toast "已切换耗材: XXX"
- **设为主挤出机**: toast
- **关闭挤出机**: toast

### 键盘快捷键
为以下快捷键绑定 keydown 事件(与菜单功能一致):
- Ctrl+N, Ctrl+O, Ctrl+S, Ctrl+Z, Ctrl+Y, Ctrl+A, Del, F11

### 弹窗/对话框
- 如果项目中已有 confirmModal,复用它
- 如果没有,创建一个通用确认弹窗(标题+文字+取消/确认按钮)
- 弹窗样式与现有 modal 一致

## 技术要求
- 只修改 app.js(如果需要 toast 容器和 file input 也可以改 index.html)
- 不引入新依赖
- Toast 组件如果不存在就创建一个
- 保持现有功能完好
